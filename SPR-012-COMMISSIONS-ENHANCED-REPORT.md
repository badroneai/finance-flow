## ٢. الميزات الجديدة

### أ. فلترة متقدمة (الخطوة 2)

| الفلتر | التفاصيل |
|--------|----------|
| **نطاق التاريخ** | حقلي `من تاريخ` و `إلى تاريخ` — يفلتر بتاريخ الاستحقاق أو الإنشاء |
| **الوكيل** | قائمة منسدلة بأسماء الوكلاء الفريدة من البيانات الموجودة |
| **الترتيب** | 5 خيارات: الأحدث، الأقدم، الأعلى مبلغاً، الأقل مبلغاً، العميل (أ-ي) |
| **عدد النتائج** | يظهر أسفل الفلاتر عند وجود نتائج |

### ب. تقارير العمولات (الخطوة 3)

تبويب "التقارير" يحتوي 3 عروض فرعية:

| التقرير | المحتوى |
|---------|---------|
| **حسب الوكيل** | تجميع بعدد الصفقات، قيمة الصفقات، العمولة، المدفوع، المتبقي |
| **حسب الدفتر** | نفس التجميع مع اسم الدفتر |
| **شهري** | تجميع بالشهر (YYYY-MM) مع عرض اسم الشهر بالعربي |

كل تقرير يتضمن:
- 3 بطاقات ملخص علوية (عدد الصفقات، إجمالي العمولات، المتبقي)
- جدول تفصيلي مع أعمدة كاملة
- **رسم بياني أفقي** (bar chart) بسيط بأشرطة CSS — الأخضر للمدفوع، الأزرق لإجمالي العمولة
- مفتاح الرسم أسفل الجدول

### ج. عرض الوكيل (الخطوة 4)

| القاعدة | التنفيذ |
|---------|---------|
| الوكيل يرى فقط عمولاته | فلتر تلقائي بـ `agentName === profile.full_name` |
| الوكيل لا يستطيع الإنشاء/التعديل/الحذف | إخفاء أزرار "إضافة"، "تعديل"، "دفعة"، "حذف" |
| الوكيل لا يرى فلتر الوكلاء | إخفاء قائمة "كل الوكلاء" |
| شريط تنبيه | رسالة زرقاء "أنت تشاهد عمولاتك فقط (وضع الوكيل)" |
| الملخص يعكس عمولات الوكيل فقط | `summary` يُحسب من البيانات المفلترة |
| التقارير تعكس عمولات الوكيل فقط | `CommissionReports` يستقبل البيانات المفلترة |

**الأدوار المدعومة:**
- `isAgent` (بدون owner/manager/superAdmin) → عرض محدود (قراءة فقط)
- `isOwner` / `isManager` / `isSuperAdmin` → عرض كامل مع CRUD

### د. تصدير CSV (الخطوة 5)

| التفاصيل | القيمة |
|----------|--------|
| الترميز | UTF-8 مع BOM (يدعم العربي في Excel) |
| الأعمدة | العميل، الوكيل، الدفتر، قيمة الصفقة، نسبة المكتب، العمولة، المدفوع، المتبقي، الحالة، تاريخ الاستحقاق، ملاحظات |
| البيانات | تُصدّر البيانات المفلترة (وليس الكل) |
| اسم الملف | `commissions_YYYY-MM-DD.csv` |
| الحالة بالعربي | مستحقة / مدفوعة جزئياً / مدفوعة |

## ٤. التفاصيل التقنية

- استخدام `useAuth()` لجلب `isAgent, isOwner, isManager, isSuperAdmin, profile`
- دالة مساعدة `f(obj, camelKey, snakeKey)` لتوحيد قراءة الحقول
- `agentOnly = isAgent && !isOwner && !isManager && !isSuperAdmin`
- `agentNames` محسوبة بـ `useMemo` من مصفوفة العمولات
- `CommissionReports` مكون مستقل يستقبل البيانات المفلترة حسب الدور
- الرسم البياني الأفقي: CSS divs مع `width: percentage%` — لا مكتبات خارجية
- تنسيق الشهر بالعربي عبر `Intl.DateTimeFormat('ar-SA', { year, month: 'long' })`
