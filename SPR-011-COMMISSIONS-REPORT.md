## ٤. كيف تعمل العمولات

### إضافة عمولة
1. المستخدم يضغط "إضافة عمولة"
2. يملأ: اسم العميل، قيمة الصفقة، نسبة المكتب (افتراضي 2.5%)
3. النظام يحسب تلقائياً:
   - عمولة المكتب = قيمة الصفقة × نسبة المكتب%
   - عمولة الوكيل = قيمة الصفقة × نسبة الوكيل% (اختياري)
   - صافي المكتب = عمولة المكتب - عمولة الوكيل
4. تُحفظ بحالة `pending` مع `paid_amount = 0`

### تعديل عمولة
- نفس النموذج مع بيانات معبأة مسبقاً
- لا يُغيّر الحالة أو المبلغ المدفوع

### الدفعات الجزئية
1. المستخدم يضغط "دفعة" على عمولة مستحقة أو جزئية
2. يرى: المبلغ الإجمالي، المدفوع سابقاً، المتبقي
3. يدخل مبلغ الدفعة (لا يتجاوز المتبقي)
4. زر "دفع كامل" يملأ المبلغ المتبقي تلقائياً
5. عند الحفظ:
   - `new_paid = paid_amount + مبلغ_الدفعة`
   - إذا `new_paid >= العمولة`: status → `paid`، paid_date يُعبّأ
   - إذا `new_paid > 0 && < العمولة`: status → `partial`
   - إذا `new_paid <= 0`: status → `pending`

### الحذف
- تأكيد قبل الحذف عبر ConfirmDialog
- يحذف نهائياً من Supabase أو localStorage
