# /project:new-feature — تخطيط ميزة جديدة

إذا حُدد اسم الميزة في $ARGUMENTS، استخدمه. وإلا اسأل.

## التعليمات

1. **فهم الميزة**:
   - اسأل: "أيش الميزة اللي تبي تضيفها؟" (إذا ما حُددت في $ARGUMENTS)
   - اسأل: "مين المستخدم المستهدف؟"
   - اسأل: "أيش المشكلة اللي تحلها؟"

2. **تحليل الجدوى**:
   - هل تناسب نطاق قيد العقار (إدارة تدفقات مالية عقارية)؟
   - هل تحتاج تبعيات جديدة في package.json؟
   - هل تحتاج جدول جديد في Supabase؟
   - كم الوقت المقدر؟

3. **تحديد الملفات**:
```
الملفات الجديدة:
  src/domain/[feature].js        — المنطق النقي
  src/pages/[Feature]Page.jsx    — الصفحة
  src/ui/[feature]/              — مكونات الواجهة (إن لزم)
  src/core/[feature]-engine.js   — المحرك (إن لزم)

التعديلات المطلوبة:
  src/App.jsx                    — إضافة route + lazy import
  src/config/navigation.js       — إضافة عنصر تنقل
```

4. **اعرض الخطة** وانتظر الموافقة قبل أي تنفيذ.
