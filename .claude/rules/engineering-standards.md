# قواعد العمل الهندسي — Engineering Standards

> آخر تحديث: 2026-04-02
> هذا الملف مرجع إلزامي — أي مخالفة تحتاج مبرر مكتوب.

## حدود حجم الملفات

| النوع | الحد الأقصى | عند التجاوز |
|-------|-----------|------------|
| صفحة (Page) | 500 سطر | تفكيك إلزامي إلى shell + مكونات |
| مكوّن UI | 300 سطر | مراجعة ضرورية — غالباً يحتاج تقسيم |
| تبويب (Tab) | 400 سطر | استخراج الأجزاء الثقيلة لمكونات فرعية |
| Core service | 500 سطر | تقسيم حسب المسؤولية |
| Domain file | 300 سطر | تقسيم حسب الكيان (إلا ملفات القوالب/البيانات) |
| Context | 400 سطر | تقسيم حسب المجال (ledger, property, contract) |

## حدود التعقيد

| المقياس | الحد الأقصى | عند التجاوز |
|---------|-----------|------------|
| useState في مكوّن واحد | 8 | استخراج custom hook أو تقسيم المكوّن |
| useEffect في مكوّن واحد | 3 | مراجعة — غالباً يحتاج فصل منطق |
| Props لمكوّن واحد | 12 | استخدام Context أو تجميع في object واحد |
| عمق التداخل (nesting) | 4 مستويات | استخراج مكوّن فرعي |

## قواعد الاستخراج

### متى يجب استخراج Custom Hook؟
- نفس النمط (useState + handler) يتكرر في 3+ ملفات
- مكوّن يحتوي أكثر من 8 useState
- منطق async معقد (fetch + loading + error) يتكرر

### متى يجب فصل Modal؟
- **دائماً** — لا modals inline في الصفحات
- كل modal في ملف منفصل تحت `src/ui/modals/` أو المجلد المناسب
- Modal يستقبل: `isOpen`, `onClose`, والبيانات اللازمة فقط

### متى يمنع تمرير Props بهذا الحجم؟
- أكثر من 12 prop = إشارة لمشكلة معمارية
- الحل: Context مخصص، أو تجميع Props في object، أو تقسيم المكوّن

## قواعد الاختبارات

| الطبقة | اختبارات إلزامية؟ | الحد الأدنى |
|--------|------------------|------------|
| `domain/` | ✅ نعم | كل دالة مُصدّرة |
| `core/` | ✅ نعم | كل دالة حرجة (calculations, engines) |
| `pages/` | ❌ اختياري | smoke test إن أمكن |
| `ui/` | ❌ اختياري | اختبار التكامل إن أمكن |

## قواعد الـ Commit

- Husky + lint-staged يمنع commit كود غير منسق
- `npm run test` يجب أن يمر قبل أي PR
- لا skip hooks بدون سبب مكتوب ومُراجع

## قواعد إضافة تبعيات (Dependencies)

- لا تبعية جديدة بدون مبرر مكتوب
- `npm audit` بعد كل إضافة — لا ثغرات critical
- تفضيل الحلول الأصلية (vanilla JS/React) على المكتبات الصغيرة
