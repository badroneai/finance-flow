# مرجع ZATCA Phase 2 — قيد العقار

## أنواع الفواتير

### الفاتورة المبسطة (Simplified Invoice — B2C)
تُستخدم للمعاملات مع الأفراد (أقل من 1,000 ر.س عادةً).

#### الحقول الإلزامية
| # | الحقل | الوصف | مثال |
|---|-------|-------|------|
| 1 | رقم الفاتورة | فريد، تسلسلي | INV-2026-00042 |
| 2 | تاريخ الإصدار | ISO 8601 | 2026-03-23T10:30:00Z |
| 3 | اسم البائع | الاسم التجاري | مكتب قيد العقار |
| 4 | الرقم الضريبي للبائع | 15 رقم | 310000000000003 |
| 5 | عنوان البائع | المدينة + الحي | الرياض، حي النرجس |
| 6 | بنود الفاتورة | وصف + كمية + سعر | إيجار شهري × 1 |
| 7 | المبلغ قبل الضريبة | بالهللة | 5000.00 SAR |
| 8 | نسبة الضريبة | ثابتة | 15% |
| 9 | مبلغ الضريبة | المبلغ × 0.15 | 750.00 SAR |
| 10 | الإجمالي شامل الضريبة | المبلغ + الضريبة | 5750.00 SAR |
| 11 | QR Code | TLV encoded | Tags 1-5 (Phase 1) أو 1-9 (Phase 2) |

### الفاتورة الضريبية (Tax Invoice — B2B)
تُستخدم للمعاملات بين المنشآت.

#### الحقول الإلزامية (إضافة على المبسطة)
| # | الحقل | الوصف |
|---|-------|-------|
| 12 | اسم المشتري | الاسم التجاري |
| 13 | الرقم الضريبي للمشتري | 15 رقم |
| 14 | عنوان المشتري | المدينة + الحي |
| 15 | تاريخ التوريد | إذا مختلف عن تاريخ الإصدار |
| 16 | رقم أمر الشراء | إن وُجد |

## TLV Tags (QR Code)

### Phase 1 — Tags 1-5 (إلزامي حالياً)
| Tag | القيمة | النوع |
|-----|--------|-------|
| 1 | اسم البائع | UTF-8 String |
| 2 | الرقم الضريبي | UTF-8 String |
| 3 | تاريخ ووقت الفاتورة | ISO 8601 |
| 4 | إجمالي الفاتورة شامل الضريبة | Decimal String |
| 5 | مبلغ الضريبة | Decimal String |

### Phase 2 — Tags 6-9 (إضافية)
| Tag | القيمة | النوع |
|-----|--------|-------|
| 6 | Hash الفاتورة | SHA-256 |
| 7 | التوقيع الرقمي | ECDSA |
| 8 | المفتاح العام | X.509 |
| 9 | ختم التشفير | CSID stamp |

### تنسيق TLV
```
كل tag = [Tag Number (1 byte)] + [Length (1 byte)] + [Value (N bytes)]
الـ QR = Base64(Tag1 + Tag2 + Tag3 + Tag4 + Tag5 + ...)
```

## قواعد الحساب

### الضريبة
```
tax_amount = taxable_amount × 0.15
total = taxable_amount + tax_amount
```

### التقريب
- لا تقريب وسيط — احسب بالدقة الكاملة
- التقريب في النتيجة النهائية فقط: أقرب 0.01 SAR (هللة)
- طريقة التقريب: Half Up (0.005 → 0.01)

### العملة
- الرمز: SAR أو ر.س
- الدقة: هللتان (0.01)
- الفاصلة: نقطة عشرية (1500.00)
- الآلاف: فاصلة (1,500.00)

---

## الرقم الضريبي
- الطول: 15 رقم بالضبط
- البداية: 3
- النهاية: 3
- التنسيق: `3XXXXXXXXXXXXX3`
- Regex: `/^3\d{13}3$/`

---

## متطلبات Phase 2

### XML (UBL 2.1)
- تنسيق: Universal Business Language 2.1
- Namespace: `urn:oasis:names:specification:ubl:schema:xsd:Invoice-2`
- كل فاتورة تحتاج UUID فريد (v4)

### التوقيع الرقمي
- الخوارزمية: ECDSA (secp256k1)
- Hash: SHA-256
- الشهادة: X.509 من ZATCA
- التوقيع يشمل: كل حقول الفاتورة

### التكامل
- API endpoint: ZATCA Fatoora Portal
- الإرسال: خلال 24 ساعة من الإصدار
- الحالات: Reported (مُبلغ) / Cleared (مُعتمد)
